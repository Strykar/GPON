# Code of conduct

This is a small open-source project; the rules are short.

## What's expected

- Be civil. Disagree with ideas, not people.
- Keep technical discussion technical. Stick to facts, repro steps, logs,
  and code; opinions are fine, attacks aren't.
- Assume good faith. Most "wrong" issues and PRs are people learning, not
  trolling. Treat them that way.
- English is the working language but it's not everyone's first; be patient
  with phrasing.

## What's not OK

- Personal attacks, harassment, slurs, threats.
- Sexual content or sexualised attention. This is a Prometheus exporter
  for a fibre transceiver; there is no context where this is appropriate.
- Doxxing or sharing other people's private information without consent.
- Sustained derailing of issues into unrelated agendas.
- Anything that would obviously belong on a "don't do this at work" list.

## If something goes wrong

If you experience or witness behaviour that violates the above, open a
private issue contact through the maintainer's GitHub profile, or email
the address listed there. Reports stay private. The maintainer will
respond, ask questions if needed, and act -- usually a conversation,
sometimes removing comments, in egregious cases banning from the repo.

The maintainer is the final call on enforcement. There is no committee.
If you disagree with an outcome, you're welcome to fork.

## Scope

This applies to everything that happens under this project: issues,
pull requests, discussions, code review comments, commit messages, and
any project-related communication elsewhere (Matrix, IRC, etc.) where
you're representing the project.

Outside the project, you do you.
